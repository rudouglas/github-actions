## Notes

A new version of the agent has been released. Follow standard procedures to [update the Infrastructure agent](https://docs.newrelic.com/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent).  
New Relic recommends that you upgrade the agent regularly and at a minimum every 3 months.

### Changed

* Integration protocol v4:
  * Log entity registration errors / warning only when debug is enabled.
  * Fix logic that checks max entity batch in bytes.
* [#224](https://github.com/newrelic/infrastructure-agent/pull/224) Allow list of infrastructure event fields restructured as deny list so that additional fields can be included.
* [#235](https://github.com/newrelic/infrastructure-agent/pull/235) Updated HTTP API to support all kind of integration payload versions, added support for exposing integrations API via raw TCP socket.
* [#268](https://github.com/newrelic/infrastructure-agent/pull/268) Update [curl version](https://curl.se/changes.html#7_74_0) to fix CVEs
* Bundled integrations:
  * Flex integration has been updated to version 1.3.7. For more information, [see the nri-flex changelog](https://github.com/newrelic/nri-flex/releases/tag/v1.3.7).
  * Docker integration has been updated to version 1.4.1. For more information, [see the nri-docker changelog](https://github.com/newrelic/nri-docker/releases/tag/v1.4.1).
* Log forwarder:
  * [#255](https://github.com/newrelic/infrastructure-agent/pull/255) Parser renamed to filter.
  * [#255](https://github.com/newrelic/infrastructure-agent/pull/253) Removed deprecated folder parameter.

### Added

* Secrets management & environment variables support:
  * [#251](https://github.com/newrelic/infrastructure-agent/pull/251) Agent configuration now supports **secrets** to manage sensitive settings (databind `variables` block). See [Secrets Management documentation](https://docs.newrelic.com/docs/integrations/host-integrations/installation/secrets-management)
  * [#260](https://github.com/newrelic/infrastructure-agent/pull/260) [#263](https://github.com/newrelic/infrastructure-agent/pull/263) [#266](https://github.com/newrelic/infrastructure-agent/pull/266) Added [environment-variables replacement](https://docs.newrelic.com/docs/infrastructure/install-infrastructure-agent/configuration/configure-infrastructure-agent) support for agent & integration configuration files.
    * Replacement of env-vars for agent config file using double curly braces `{{ ENV_VAR }}`.
    * Env-vars replacement happens before YAML is parsed or _databind_ (secrets/variables engine) runs.
  * [#258](https://github.com/newrelic/infrastructure-agent/pull/258) Added [New Relic CLI](https://github.com/newrelic/newrelic-cli/blob/c41108a2f47493e05542ce02e87f1225a72e7f76/docs/cli/newrelic_agent_config_obfuscate.md) `obfuscated` support for secrets. See [obfuscated provider](https://docs.newrelic.com/docs/integrations/host-integrations/installation/secrets-management#newrelic-cli-obfuscation) in Secrets management documentation.
