## Notes

A new version of the agent has been released. Follow standard procedures to [update your infrastructure agent](https://docs.newrelic.com/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent).

### Added

* [#53](https://github.com/newrelic/infrastructure-agent/pull/53) Dimensional metrics are now processed by the agent, registering entities for them. This feature is disabled under a feature-flag.
* [#90](https://github.com/newrelic/infrastructure-agent/pull/90) The log forwarder now captures the name of the file being tailed as Path_Key.
* [#96](https://github.com/newrelic/infrastructure-agent/pull/96) The agent now supports accessing secrets from CyberArk either using an HTTP endpoint or the CLI tool.

For more information on secrets management using CyberArk, [see our docs](https://docs.newrelic.com/docs/integrations/host-integrations/installation/secrets-management#cyberark-cli).

### Fixed

* [#101](https://github.com/newrelic/infrastructure-agent/issues/101) Fixed issue where the agent would panic when trying to compact inventory data.

### Changed

* [#109](https://github.com/newrelic/infrastructure-agent/pull/109) The built-in [Flex integration](https://github.com/newrelic/nri-flex) has been updated to version 1.3.5. For more information, [see the Flex changelog](https://github.com/newrelic/nri-flex/releases/tag/v1.3.5).
* [#108](https://github.com/newrelic/infrastructure-agent/pull/108) curl has been updated to 7.69.1-r1 in the base image.
