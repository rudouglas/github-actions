## Notes

A new version of the agent has been released. Follow standard procedures to [update the Infrastructure agent](https://docs.newrelic.com/docs/infrastructure/install-configure-manage-infrastructure/update-or-uninstall/update-infrastructure-agent).

### Added

* [#172](https://github.com/newrelic/infrastructure-agent/issues/172) The Infrastructure agent now includes the nri-prometheus [v2.2.1](https://github.com/newrelic/nri-prometheus/releases/tag/v2.2.1) integration embedded with Linux packages and Docker image. For more information about the Prometheus integration, see our [docs](https://docs.newrelic.com/docs/integrations/prometheus-integrations/get-started/send-prometheus-metric-data-new-relic)
* [#152](https://github.com/newrelic/infrastructure-agent/issues/152) Integrations with protocol v4 are now being registered and stored in memory by their unique entity name.
* [#125](https://github.com/newrelic/infrastructure-agent/issues/125) Improvements in protocol v4 dimensional metric submission.
* [#174](https://github.com/newrelic/infrastructure-agent/pull/174) Optional list of CLI arguments when configuration `name` is used instead of `exec`.

### Fixed

* [#177](https://github.com/newrelic/infrastructure-agent/issues/177) The Infrastructure agent now properly handles nested inventory items.
