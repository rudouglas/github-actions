Logs in context for the Java agent connects your logs and APM data in New Relic, giving full context to high-level events and providing high value data to specific log lines. Read on to learn how to configure logs in context and enrich your log data.

## Compatibility and requirements [#compatibility-requirements]

To use log management with the Java agent, ensure your configuration meets the following requirements:

* [Log management](/docs/introduction-new-relic-logs) enabled, with a compatible [log forwarding plugin](https://docs.newrelic.com/docs/logs/enable-logs/enable-logs/enable-new-relic-logs#enable-logs) installed
* Java agent 5.6.0 or higher: [Install](/docs/agents/java-agent/installation/install-java-agent) or [update](/docs/agents/java-agent/installation/update-java-agent)

## Configure logs in context for Java [#enable-logs-java]

Choose an extension to see specific instructions:

* [Dropwizard v1.3 extension](/docs/enable-logs-context-dropwizard)
* [java.util.logging extension](/docs/enable-logs-context-java-util-logging)
* [Logback extension](/docs/enable-logs-context-logback)
* [Log4j 1.x extension](/docs/enable-logs-context-log4j-1x)
* [Log4j 2.x extension](/docs/enable-logs-context-log4j-2x)

Use Spring or Spring Boot and not sure which extension you need? See our [Spring documentation](/docs/logs/enable-new-relic-logs/logs-context-java/configure-logs-spring-spring-boot).

## What's next? [#what-next]

Now that you've set up APM logs in context, here are some potential next steps:

* Troubleshoot errors with [distributed tracing](/docs/understand-dependencies/distributed-tracing/get-started/introduction-distributed-tracing), stack traces, application logs, and more.
* [Query your data](/docs/using-new-relic/data/understand-data/query-new-relic-data) and [create custom dashboards](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards) or [alerts](/docs/alerts/new-relic-alerts/configuring-alert-policies/create-edit-or-find-alert-policy).
