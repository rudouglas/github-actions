The New Relic Infrastructure [Integrations SDK](/docs/infrastructure/integrations/intro-infrastructure-integrations) lets you create an [on-host integration](/docs/integrations/host-integrations/getting-started/introduction-host-integrations) for reporting custom host and service data. This document explains what files an on-host integration requires. New Relic also provides [Go language integration building tools and a tutorial](/docs/infrastructure/integrations-sdk/tools-resources/infrastructure-integration-building-resources).

## Integration files [#overview]

Before you build an on-host integration, verify you meet the [compatibility and requirements](/docs/integrations/integrations-sdk/getting-started/compatibility-requirements-infrastructure-integrations-sdk).

An integration requires at least these files:

* An [executable file](/docs/infrastructure/infrastructure-integrations/custom-integrations/create-integration-executable-file), written in any language, that export JSON data in a format expected by the Infrastructure agent
* One or more [configuration files](/docs/integrations/integrations-sdk/file-specifications/host-integrations-standard-configuration-format)

For Go language build tools and a tutorial for creating these files, see [Build resources](/docs/integrations/integrations-sdk/getting-started/integrations-tutorials-build-resources).
